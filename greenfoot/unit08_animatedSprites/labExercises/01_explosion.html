In this lesson we introduce a new class named AnimatedActor. AnimatedActor enables actors to be animated by allowing them to have multiple images assigned to them. AnimatedActors retrieve their images from sprite sheets.

To demonstrate how the AnimatedActor class works let's walk through the steps needed to animate a series of explosion images.

AnimatedActor Class

Double-click the AnimatedActor class to view its code. Familiarize yourself with the methods that are provided by this class. The AnimatedActor class was written by me and is not part of the Greenfoot environment so it must be added to projects that want to use it.

Sprite Sheet

A sprite sheet is a file that contains a series of images with each image having the same dimensions. Placing all of the images into one file makes it easier to load the images into a program. Look at the sprite sheet for an explosion. (The image has been reduce in size so that it will fit on this page.)

Each image on the sprite sheet must be the same size. For example all 13 images on this sheet have dimensions of 196 x 190.

Explosion Class

Double-click the Explosion class to view its code. Look at the Explosion class constructor shown below.

The constructor includes 4 parameters used in creating an AnimatedActor.

  • imageFile - the name of the sprite sheet file

  • numFrames - the number of images(frames) contained in the sprite sheet

  • frameWidth - the width of each frame in the sheet

  • frameHeight - the height of each frame in the sheet

Every class that is a subclass of AnimatedActor must include this four parameter constructor and the first line of the constructor must include the following line of code.

super(imageFile, numframes, frameWidth, frameHeight);

The keyword super is used to pass the information about the sprite sheet onto the superclass (AnimatedActor) so that it can be used by the AnimatedActor class to animate the actor.

MyWorld Class

The MyWorld class is where the Explosion object is created. Several AnimatedActor methods are called to do some initial setup for the animation. Let's look at this code line by line.

explosion = new Explosion("explosion_sheet.png", 13, 196, 190);

This line of code creates the Explosion object. "explosion_sheet.png" is the name of the sprite sheet. 13 is the number of frames in the sprite sheet, 196 is the width of each image(frame) in the sheet, and 190 is the height of each frame in the sheet.

addObject(explosion, getWidth()/2, getHeight()/2);

This code adds the Explosion object to the world positioned at its center.

explosion.setAutoAdvance(true);

The method setAutoAdvance turns on or off the auto advance feature. When set to "true" this method enables the frames to be cycled through automatically which allows the actor to be animated.

explosion.setAdvanceSpeed(10);

The method setAdvanceSpeed sets the animation speed for the actor. The larger the number passed as a parameter the slower the animation will advance through its frames. A value of zero represents the fastest animation speed.

explosion.setLooping(true);

The method setLooping turns on or off the looping feature. If looping is set to "true" when the animation reaches its last frame it advances back to the first frame and starts the animation over again.

Program Modifications

  1. If you have not already done so compile and run the program.

  2. Modify the animation speed by setting it to 0. Compile and run the program.

  3. Modify the animation speed by setting it to 100. Compile and run the program.

  4. Set the looping feature to false. Compile and run the program.

  5. Set the auto advance feature to false. Compile and run the program.

  6. Add the following act method to the Explosion class.

    public void act()
 {
     super.act();  // call AnimatedSprite's act method
        
     String key = Greenfoot.getKey();
     if("right".equals(key))
     {
         setFrame(getFrame() + 1);  // advance to next frame
     }
}

    Notice the command super.act(). This line of code is required and must be the first line of the act method.

    Compile and run the program. Use the right arrow key to advance the frames.

  7. Turn the looping feature on then compile and run the program.
