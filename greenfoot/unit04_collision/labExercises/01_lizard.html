Collision Detection

In gaming, a collision is the act of one object intersecting with another object. An example of a collision would be a bullet hitting its target. Collision detection refers to the process of detecting collisions. Greenfoot provides several methods in the Actor class that can be used for collision detection.

  1. Click on the Help menu and select Greenfoot Documentation.

  2. On the left side of the document select Actor to view the documentation for the Actor class. Scroll down to the section Method Summary and look for methods that can be used for collision detection.

  3. What is the difference between the methods isTouching, intersects, and getOneIntersectingObject?

Method isTouching

The isTouching method returns true when an Actor of the specified class intersects with the Actor object doing the test, otherwise it returns false. Look at the following example.

if(isTouching(Bug.class))
{   
    // code that responds to collision
}

The if statement is true when this object(the class where this code resides) is touching any Bug object.

Method intersects

The interects method returns true when a specific Actor intersects with the Actor doing the test, otherwise it returns false. Look at the following example.

if(intersect(ladyBug))
{   
    // code that responds to collision
}

The if statement is true when this object is touching the ladyBug object.

Method getOneIntersectingObject

The getOneIntersectingObject method returns the Actor of the specified class that is currently intersecting with the object doing the test. If no Actor is currently intersecting it returns null. Look at the following example.

Actor actor = getOneIntersectingObject(Bug.class);  // get colliding actor
if(actor != null)   // if collision occurred
{   
    // code that responds to collision
}

The if statement is true when actor contains a reference to a Bug object.

In summary, both the isTouching and interests methods return a boolean value of true or false, whereas the getOneIntersectingObject method actually returns that Actor that did the colliding.

Lizard class

Open the Lizard class to view its code.

The Lizard class contains two methods named spinLizard and moveLizard. The spinLizard method will cause the lizard to spin around in a complete circle and then remove any flies it is touching. The moveLizard method will move the lizard around the world when the left, right, up, or down arrows have been pressed.


Program Modifications

  1. Complete the Lizard class's act method so that it when the lizard collides with a Fly it spins around in a circle and eats the fly. If the lizard has not collided with a Fly it should move if the left, right, up, or down arrow keys have been pressed.
