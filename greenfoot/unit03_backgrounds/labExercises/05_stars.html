Stars

In this lesson we are going to create a space scenario with stars that move on a black background. However instead of creating a scrolling background by moving a background image we are going to create a more dynamic background by moving Star actors on top of the black background. This will give us more flexibility by being able move the stars with different speeds and different levels of brightness.

Star Class

Double-click on the Star class to view its code.

The Star class does not include an image file because its image will be created when the star is added to the world.

Instance variables

The Star class includes two instance variables. The SIZE variable represents the size of the image. Notice the use of the keyword final in its declaration.

private 
final
 int SIZE = 3;   // size of star

The final keyword turns a variable into a constant. A constant's value cannot be changed once it has been assigned a value. By convention, Java programmers typically declare constants using all uppercase letters. This makes them easily identifiable within your code. Technically using constants within your code is not a requirement. Programmers use them to make the code more readable and some compilers will optimize the code when they encounter them.

The speed variable is used to store the speed of each individual star.

Constructor

The Star constructor contains code that will be executed when a Star object is created. Study the following code then add it to the constructor.

GreenfootImage image = new GreenfootImage(SIZE, SIZE);   // create new image
        
int c = Greenfoot.getRandomNumber(255);  // random color value
image.setColor(new Color(c, c, c));      // set color to shade of gray
image.fillOval(0, 0, SIZE, SIZE);        // draw star
        
speed = Greenfoot.getRandomNumber(5) + 1;  // assign a random speed
        
setImage(image);      // set star image to image we just created above

Method act

The act method contains code that will be executed continually while the program is running. The act method moves the Star object toward the left edge of the world. It monitors the star's location so that when it reaches the left edge of the world it picks it up and moves it to the right edge of the world at a random vertical location. Study the following code then add it to the act method.

setLocation(getX() - speed, getY());     // update stars location
        
if(getX() < SIZE)  // if star reaches left edge of world
{
    int x = getWorld().getWidth() - getX() - SIZE;   // move star to right edge
    int y =  Greenfoot.getRandomNumber(getWorld().getHeight());  // at random y position
    setLocation(x, y);  // move star
}

Space Class

Double-click on the Space class to view its code.

Constructor

The Space constructor contains code that will be executed when the world is created. Study the code below then added it to the constructor.

setPaintOrder(Rocket.class, Star.class);   // make rocket appear on top of stars
        
getBackground().setColor(Color.black);    // set Background color to black
getBackground().fill();
        
Rocket rocket = new Rocket();
addObject(rocket, getWidth()/2, getHeight()/2);   // add rocket
        
addStars(100);   // add a hundred stars to world

This code introduces a new World method called setPaintOrder. 

setPaintOrder(Rocket.class, Star.class);   // make rocket appear on top of stars

This method sets the paint order of objects within a world. In the declaration the Rocket class is listed first so it will be painted on top of any Star objects it passes over. All Star objects will pass behind the Rocket as they move across the screen.

Method addStars

The method addStars creates Star objects and adds them to the world at random locations. The parameter howMany determines how many stars are added to the world. In the constructor the following call is made to the addStars method:

addStars(
100
);   // add a hundred stars to world

The value 100 is called an actual parameter. It is the value assigned to the parameter howMany when the method is executed.
