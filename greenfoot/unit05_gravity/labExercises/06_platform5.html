Release Shark

This version of the program modifies the way the game ends. Instead of just ending the game when the frog lands on the ground like previous versions, this version releases a shark to eat the frog when it lands on the ground.

Look at the Frog method endGameTest.

When the frog touches the ground a Shark object is created and added to the world. The shark's initial position is off the screen to the right.

Notice the use of the boolean variable releaseShark in the if statement. This variable, called a flag, is necessary to prevent more than one shark from being created each time the endGameTest method is called. Recall that the act method, and any code placed inside the act method, is executed over and over again as the program is running. Without this flag in the if statement the code that creates a Shark object would be called repeatedly as long as the frog is touching the ground resulting in numerous sharks being added to the world.

The act method in the Shark class causes the shark to swim across the world from right to left.


Program Modifications

  1. Add a method named sharkTest to the Frog class. This method, when called, should test to to see if the shark is touching the frog. If the shark is touching the frog the frog should be placed in the shark's mouth and carried off the screen.

    Use the following pseudocode:

    if frog is touching a shark then
   get reference to the shark
   move frog to shark's location
end if

    The following code can be used to get a reference to the shark and move the frog to the shark's location: 

    Actor shark = getOneIntersectingObject(Shark.class);     // get the shark
setLocation(shark.getX(), shark.getY());     // move frog to shark's location

  2. Make a call to the sharkTest method in the act method.

  3. Compile and run the program. Notice that the frog is positioned in the center of the shark.

  4. Adjust the frog's location so that the frog is positioned inside the shark's mouth.
