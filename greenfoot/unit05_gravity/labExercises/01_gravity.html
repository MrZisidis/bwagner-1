Linear Motion

Add a Ball1 object to the top of the world and press the Run button.

Ball1 is an example of linear motion. Linear motion is movement that occurs at a constant rate. Look at the code for the Ball1 class.

The class defines a variable named dy that represents the ball's vertical velocity. Linear movement is created by updating the ball's vertical position by dy each time the act method is executed. Since dy has a value of 2 the ball's vertical position changes by 2 units each time it moves.

Initial Velocity

Add a Ball3 object to the bottom of the world and press the Run button.

The Ball3 class defines a variable named initialVelocity. This value is assigned to dy before the ball moves. By starting dy with a negative value the ball's movement is upward, but as the force of gravity is continually applied to dy the ball eventually begins to move down as dy's value becomes positive.

Add the following code to Ball3's act method:

if(getY() >= getWorld().getHeight() && dy > 0)  // if at bottom of world
{
    initialVelocity = initialVelocity * .8;   // decrease by 20%
    dy = (int)initialVelocity;    // cast initialVelocity to an int      
}

This code causes Ball3 to continually bounce up and down until it eventually comes to rest. Each bounce is calculated to be 20% less than the previous bounce. Notice that the variable initialVelocity was declared a double so that the calculation for reduction in bounce could be achieved.

In Ball5 all of the variables are declared double which allows us to create more precise movements.

Change some of the variables values and watch how the ball's movement is effected.

Casting

Look at the following line of code from Ball5's act method:

setLocation(
(int)
(getX() + dx), 
(int)
(getY() + dy));

An object's location within a world is always stored as a pair of integer values (x, y). Since our dx and dy values are doubles the results of the calculations inside the parentheses must be cast to integers before they can be passed to the setLocation method.
