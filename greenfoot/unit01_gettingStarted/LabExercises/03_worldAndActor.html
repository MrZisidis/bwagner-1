World and Actor

Greenfoot Classes

The Greenfoot environment consists of seven core classes:

  • Actor
  • Greenfoot
  • GreenfootImage
  • GreenfootSound
  • MouseInfo
  • UserInfo
  • World

These classes are specifically designed to make writing games and graphical simulations easier. In this assignment you are introduced to the World and Actor classes.

Directions

In Greenfoot a scenario is a computer program consisting of Java classes, images, and sounds. Follow the steps below to create your first Greenfoot scenario.

    Initial Setup

  1. Navigate to your Home Directory(H: Drive) and create a new folder named MyGreenfoot.
  2. Click on the Greenfoot icon to open the Greenfoot Environment.
  3. Select Scenario --> New
  4. Navigate to the MyGreenfoot folder you created in step 1, type MyFirst, and click the create button.

    The following screen should appear:

    5. Creating a World

    Once a scenario has been created the next step is to create a world. A world in Greenfoot is a two-dimensional grid that you can place "actors" on. From a programming perspective a world is a class that contains variables and methods. Double-click on the World button to see the methods defined in the World class.

  5. Right-clicking on the World button brings up the folowing menu:


    Select --> New Subclass...
  6. In the New Class box type MyWorld and click Ok.

    You should now see a subclass named MyWorld listed below the World class.

    What is a subclass and why did we make one?

    Because of the way the World class was designed it cannot be modified. That means no code can be added or removed. However to make our own programs do new and different things we need to be able to add our own code to this class. In Java, this can be accomplished by making a subclass. A subclass is a class that is a copy of another class with the new class having the same capabilities as the original class (i.e. same methods), but with the added benefit of being able to add new code.

  7. Double-click on MyWorld to see your new class.

    Notice the use of the keyword extends on the line of code that defines the MyWorld class. The process of one class extending another class is called inheritance. Inheritance is one of the most important features of object-oriented programming because it allows a class to inherit all of the capabilities of another class plus include additional code that extends the new class's capabilities beyond that of its parent class. The importance of this feature will become more apparent as we explore Greenfoot further.

    8. Creating an Actor

  8. The next step in building a Greenfoot scenario is to add actors to your new world. Greenfoot provides an Actor class for this purpose. Double-click on the Actor button to see the methods defined in the Actor class.

  9. Like the World class, the Actor class cannot be modified so we must create a new class that is a subclass of the Actor class. To create a new subclass right-click on the Actor button and select New subclass...

  10. Give the new subclass the name Bug.

  11. Greenfoot allows you to assign a graphic to an Actor subclass. Under Images Categories: select animals then scroll down the list on the right until you see the image ladybug_02. Select the image and click Ok.

    12. Adding Code to Bug Class

  12. Double-click on the Bug button to see the Bug class.

  13. Notice that the Bug class extends the Actor class. This means a Bug can do everything an Actor can do. For example, an Actor can do things like move and turn. What's cool is that our Bug class can now can do these things as well.

  14. One of the most important methods the Bug class inherits from the Actor class is the act method.

    The act method is important because when you execute a scenario the act method is called repeatedly the entire time the program is running. Any code placed in the act method will be executed many times every second. This is what makes animations and games possible.

    The act method, as it is written in the Actor class, doesn't really do anything useful, therefore in our Bug class we are going to need to add some of our own code to this method. In programming terminology, we say we are overriding the act method.

    To demonstrate how this works add the following line of code to the act method:

    move(1);

    Each time the act method is called our Bug will move one unit ahead. Remember our Bug knows how to move because it inherited this behavior from the Actor class.

  15. Press the close button.

    16. Compile Scenario

  16. Before you can execute a scenario you must compile it. Press the compile button.

    17. Place Bug in the World

  17. So far you have created a Bug class and added code to make a bug move. The next step is to add a bug to the world. To do this you need to create a Bug object, using the Bug class, and then add this Bug object to the world.

    There are three ways you can add an Actor to the world. Use each technique to add a Bug object to the world.

    Method 1

    • Right-click mouse on Bug class

    • Select New Bug()

    • Use the mouse to place the Bug object in the world.

    Method 2

    • Click the mouse on the Bug class to select it.

    • Hold down the shift key. A Bug object will appear.

    • Drag the Bug object to the world and click the left mouse button to place it in the world.

    Method 3

    • Double click on the MyWorld class to view its code.

    • Inside the MyWorld constructor place the following code:

      Bug bug = new Bug();                            // create bug object
addObject(bug, getWidth()/2, getHeight()/2);    // add bug object to world

      This code will place a Bug object in the center of the world.

    • To see the Bug you must compile the program.

    18. Execute Scenario

  18. Click the Act button to execute the Act method one time.

  19. Click the Run button to execute the Act method repeatedly.

    20. Create Your Own SubClass

  20. Create a subclass of the Actor class. Give it a name and assign it an image.

  21. Create an object of your new Actor subclass using Method 3 and add it to the world as described above.

  22. Compile and execute the scenario.

  23. Use Method 3 to add a second object of your new Actor subclass to the world at a different location.

    For example, the following code would add a new Bug object at location (100, 200):

    Bug bug2 = new Bug();     // create bug object
addObject(bug2, 100, 200);     // add bug object to world

  24. Compile and execute the scenario.
