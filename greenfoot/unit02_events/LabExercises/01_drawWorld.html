DrawWorld class

The purpose of the DrawWorld class is demonstrate how to check for mouse events in a program. Mouse events occur when a user moves the mouse, clicks a mouse button, or drags the mouse while a button is pressed.

  1. Click on the Help menu and select Greenfoot Documentation.

  2. On the left side of the document select Greenfoot to view the documentation for the Greenfoot class. Scroll down to the section Method Summary to view the list of methods the class provides.

  3. Go back to the Greenfoot environment. Compile and run the Draw Scenario. While the program is running click the left mouse button and drag the mouse around the world. What happens when you click the right mouse button?

  4. Click on the DrawWorld class to view its code. Notice the use of the Greenfoot class methods mousePressed, mouseDragged, and mouseClicked.

  5. What is the difference between a mousePressed event and a mouseClicked event? Go back to the Greenfoot class documentation to find out.

  6. Notice the use of the class MouseInfo. What is its purpose? Go to its documentation to find out.

  7. Study the code for the DrawWorld class to see if you understand how it works. Discuss it with your neighbor.

Program Modifications

The World class provides a method named showText. This method will display a string in the world at the specified location (x, y). For example, the following code will display the string "Hello" at location (50, 100):

showText("Hello", 50, 100);

  1. Modify the act method in the DrawWorld class so that when the user presses the left mouse button the location of the mouse click will appear on the screen in the following format: (x,y) where x is the x-coordinate of the mouse when the button was pressed and y is the y-coordinate of the mouse when the button was pressed.

    To accomplish this task do the following:

    • Decide where the new code should be placed in the DrawWorld act method. The act method currently uses three Greenfoot class methods: mousePressed, mouseClicked, and mouseDragged. The mousePressed method returns true if a mouse button has been pressed. The mouseClicked method returns true if a mouse button has been pressed and released. The mouseDragged method returns true if the mouse is being moved while holding down a mouse button. Which if statement should your new code be placed? Right, the mousedPressed method.

    • Create a string representation of the mouse's location when pressed. Look at the following code from the DrawWorld act method

      if(Greenfoot.mousePressed(this) == true)
{
   greenfoot.MouseInfo info = Greenfoot.getMouseInfo();  // get info about mouse
   int button = info.getButton();                        // get button press info
            
   if(button == 1)  // 1 = left button was pressed
   {
      startX = info.getX();
      startY = info.getY();
   }
}

      When the left mouse button is pressed this code stores the location of the mouse in the two variables startX and startY. Therefore you need to create a string using these two values. The following code creates a string named str that stores the mouse's location in the following format: (x, y).

      String str = "(" + startX + "," + startY + ")";

      Add this code to the act method.

    • The last step is use the showText method to display the string in the world.

      showText(str, startX, startY);

      Add this code to the act method.

  2. The Greenfoot class provides another method named mouseDragEnded that returns true when the left mouse button is released to mark the end of a mouse drag event. Look at the following code segment

    if(Greenfoot.mouseDragEnded(this) == true)
{
   greenfoot.MouseInfo info = Greenfoot.getMouseInfo();
   int button = info.getButton();
            
   if(button == 1)  // 1 = left button
   {
      int endX = info.getX();
      int endY = info.getY();
   }
}

    When a drag event has ended this code stores the location of the mouse in the two variables endX and endY.

    Add the code above to the act method then modify the code so that the coordinates of the mouse's location are displayed when the user stops a mouse drag event. The coordinates displayed should use the same format as above.
