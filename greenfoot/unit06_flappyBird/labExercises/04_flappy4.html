In this version of the program a scoreboard has been added to the world. When FlappyBird passes through a set of pipes the scoreboard is increased by one. The scoreboard does not appear until Flappy reaches the first set of pipes.

Scoreboard Class

The following Scoreboard class is used to create a ScoreBoard object.

An image is not initially assigned to the Scoreboard class in the Greenfoot environment. The image is instead created in the Scoreboard's constructor. The image is created as a blank image with dimensions 150 x 80.

The method setScore is used to update the score when Flappy passes through a set of pipes. The method follows these steps to update the scoreboard:

  • Gets a reference to the image.

  • Clears the image so it is blank.

  • Selects the font attributes used to draw on the image.

  • Draws the value of the parameter score onto the image.

FlappyWorld Class

A Scoreboard object is created, added to the world, and updated in the FlappyWorld class.

Look at the method displayScore.

This method is responsible for increasing the score and calling the ScoreBoard method setScore to update the scoreboard with the new score.

The act method calls the displayScore method each time Flappy passes through a set of pipes. The tricky part of this code is knowing when Flappy passes through the pipes. This is accomplished by adding a couple of counter variables to the FlappyWorld class. Recall from the previous lesson that by using a counter variable and an if statement we were able to space the pipes so that they were always 100 units apart. Knowing this fact, namely that the pipes are always 100 units apart, all we have to do is count the number of "steps" Flappy takes. In this lesson I decided to use a different approach. Instead of counting to 100 and then resetting the counter back to zero I choose to check to see if the counter is divisible by 100. By doing it this way it made the rest of the code easier to implement. Notice that I also changed the name of the counter variable to pipeCounter.

It is the next two if statements that help us determine when Flappy passes through a set of pipes. I did some test runs and determined that Flappy reaches the first set of pipes when pipeCounter equals 210. I stored this value in a constant named FIRST_PIPE. The first if statement is used to determine when Flappy reaches the first pipes. From that point on we start counting Flappy's steps by incrementing a variable named flappyCounter. When the second if statement is true we know that Flappy has reached the next set of pipes so the displayScore method is called to update the scoreboard.
